负载均衡

其实就是专门搞了一个反向代理，然后进行请求转发，可以通过 http，也可以通过 fastcgi 技术。

也可以额用 upstream 实现调度，负载均衡，下面写的很清楚了：

Nginx 负载均衡服务器： IP：192.168.0.4（Nginx-Server）
Web服务器列表：

  • Web1:192.168.0.5（Nginx-Node1/Nginx-Web1）
  • Web2:192.168.0.7（Nginx-Node2/Nginx-Web2）

实现目的：用户访问 Nginx-Server（ http://192.168.0.4:8888 ）时，负载均衡到 Web1 和 Web2 服务器

配置 nginx 负载均衡服务器:

proxy_pass 到 webhost，然后 upstream 设置了 webhost 的主机。
上面的调度算法采用一种即可，比如 ip_hash 。

注意：这里的 webhost 可以是 ip 或者域名

每次访问 192.168.0.4:8888 这个站点的时候，Nginx会根据来访的主机的 ip_hash 值，负载均衡到 192.168.0.5（Nginx-Node1/Nginx-Web1）和 192.168.0.7（Nginx-Node2/Nginx-Web2）服务器上。如果其中一个Web服务器无效后，负载均衡服务器会自动将请求转发到正常的Web服务器。

上面是三个容器的情况，还有一种方式，如果只配置一台 nginx 容器，
那么配置到同一网段的不同虚拟主机或者不同端口，例如下面用虚拟主机配置到不同端口:

实际上配置一台容器，又是同一个 ip 主机，那么根本不用 proxy_pass 转发，直接多个不同域名的 server 都监听80好了。
因为之后大家要响应的 root 页面地址是不一样的，所以也能区分好，只是问题是，同时访问这两个域名的时候，都要这一个容器做判断，压力增长速度快。
