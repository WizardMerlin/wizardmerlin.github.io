谈谈后端开发
post time: 2017-05-26
思考

如果你只是说的 Web 时代的后台开发, 那么多半会碰到前端 JSP，Java，JS，CSS，HTML 等, 后端 Java, C++等. 实际上, 很多招聘人员自己根本分不清.

为何有此一文

总在群里被问道, 各种烦人的 学这个有前途嘛?, 我要不要学?…

其实不是烦, 因为我也经历过这个时期, 但是怎么说呢? 就我现在的观点来看:

不是你要去学什么? 而是你进了哪一行, 并且决心做下去, 那么这一行需要啥, 你都要学; 而且尽量做到最好.

用心概述

不过在现在这个全面移动时代, 全面云时代, 半智能时代, 后台开发已经变得非常非常的庞大了, 不在仅仅只是Web层面, 包含更多的内容, 但是根本性的几个东西没有变:

  • 服务器
  • 网络
  • 存储(包括数据库, 分布式)
  • 公司的业务

是的, 根据工作的领域, 书写的业务代码也不一样.在腾讯, 我了解到同学说后端(台)开发和服务器开发是一个意思, 但是显然后端的范畴大过服务器:

  • 一般后台开发指的是运行在服务器的程序，比如java/php/.net等等．你只需要会写java/php等代码就行
  • 服务器端开发可能是其他的高性能数据处理，分部式应用等等，这就需要有linux, c/c++ ,tcp/ip等知识才行

其实主要看你所在的公司的业务领域, 游戏?数据分析?专注存储?还是搞金融?

就我的实际体验(面过的单位), 越是薪水高的, 对你解决问题的能力要求越高(而不是某个知识点技能点)

Linux+Cpp

这种后台其实一句话就能概括: 在Linux下进行 C++ 网络开发(IO, IPC, TCP), 数据库操作.

一般不会让你用 C++ 写Web, 但是即便要你写, 了解一下HTPP(get/post), cookie/session, CGI等技术也就差不多了.

这里不展开了, 虽说就是普通的网络编程, 但是一旦展开内容不少. 有人认为很难? 其实也没有, 比如我问你

muxtex和spin_lock的区别?
dynamic_cast怎么实现呢?

我没有展开, 是怕吓到你(可能2-3年内都非常辛苦); 如果我有文章写面经, 你就知道要学很多东西了, 哦关键还是理解&吸收&转换;

特别如果你在金融业, 那么对你的数值运算代码, 主要是浮点型; 实时代码要求比较多.

Java+Spring?

这个里面的轮子是最多的, 各种框架满天飞. 如果说C/C++各种库的依赖搞得烦死人, 那么在这里你就会遇到各种jar包版本混乱, 不兼容等问题.

在这里, 你想处理海量数据, 跑很多服务器, 呵呵. 因为java简单, 所以会更加偏业务, 所以如果不是BAT或者fotune 500之类的, 对你的要求也没有那么高. 我有同学就是天天在写业务代码.

但是这并不表示你要学的东西少, 相反, 由于框架多, 在初期要的东西并不比C++那一波少.

现在Java后端发展到现在, 已经不是简单的spring或者spring mvc了, 云时代的要求, 可能会让你去写map reduce程序, 用Hbase等.

但是, 基本上你是离不开Web的范畴的.

LAMP

这个基本就是Web后台开发了, 服务端开发，特别是Web开发，基本上全是处理HTTP请求的处理。根据具体用途分为两种：Web页面开发和API接口开发。Web页面开发也完全可以看成是API接口开发，只是它的两个主要部分，页面和ajax请求，一个是返回html，另外一个可以返回html，也可以返回其他格式的而已。API接口开发是针对有客户端产品而言的。可能是移动设备，可能是PC应用等。

应用框架一般使用的是LNMP或者LAMP，基本的框架就是前端N台Web服务机 + cgi访问PHP + php访问mysql。

web服务器现在nginx是越来越多使用，nginx比较apache的优势就在于轻便和静态页面的高并发性能。一般拿到设备先需要考虑下单机可承受的qps大概多少，方法大致就是先只考虑内存，计算同时能开启多少个php-cgi，比如一个4G内存的机器，每个php-fpm大概占用20M内存，所以差不多能开启200个php-cgi进程（一般会留些空余的），每个进程同一个时间只能跑一个php程序，所以假设每个php程序跑0.1s，1s就能处理10个请求，所以单机qps大概会是2000。当然，一般不会开启到这么极致的程度，有几个原因：
1 需要考虑到其他进程使用内存的情况
2 考虑到如果一旦全部内存都使用完了，是否启用swap，如果没有的话，那机器是否就立即当机
3 还需要考虑到CPU和带宽的使用情况。CPU对一些比如加解密，视频转码等操作比较耗时，这个时候如果没有使用队列，那么每个请求的时间就会加长。

其他的还要涉及到文件服务器, web页面, API开发, 当然还有一些列用户要求的非功能需求开发, 比如调用统计, 日志处理等.

其他不入流的

我有点名说Golang么?

现在我身边的人对于Go语言褒贬不一, 有人认为是后端神器, 有人觉得就是Google的小组语言. 我个人的观点: 看你公司怎么来.

如果你是负责人, 要你选: 选你熟悉的语言.

(据说Go把许多后端代码进行很好的简化, 还引入了协程, 很神奇?)

日常工作

分工与细化

大公司的分工就比较细了, 可能你只负责Restful Api , 存储, 日志 等某一块. 对个人来说，服务器端开发却使终是一块迷雾环绕的大山，在这座大山上，你对你走过的路线很熟悉，而那些你没有接触过的服务器端技术，你使终是不清楚的。

从这一点来说，去创业型的公司做服务器端开发又要比去大公司要好得多。

半夜上线

恐怕会有人觉得我在开玩笑

7x24的服务, 绝对不是开玩笑; 如果服务中断, 或者在夜里出了什么大问题, 比如晚上12点上网高峰, 那么一般会在夜里2-3点进行维护.

如果碰上大升级, 要上线, 可能要通宵加班, 我这里引用网友的描述:

上线出问题怎么办？在网易，我们一般是两种做法：

在线调试和修改 Bug，通常顺利的话，到早上 7，8 点的时候一般就能搞定了。
在线调试和修改 Bug，如果不顺利的话，到早上 7，8 点的时候回滚数据。
然后，我们需要回家休息半天之后，继续在下一个凌晨 3 点上线。

立即响应

服务器端的问题，只要出现，影响都是巨大的，所以服务器端的同学需要随时待命，一旦有服务器异常报警，就立即处理，以免影响到用户。所以做服务器开 发的同学手机都是开着的，在服务不稳定时，一不小心，几十条报警短信就过来了。服务器开发出去旅游什么的，一般都带着笔记本，以便出问题时能够及时响应。

不管有多少客户端&多少种类的, 一般公司一种类型的服务器.

薪资待遇

从纯技术上的挑战来说，服务器端明显更大。服务器端开发动辄超过 10 人的团队，代码量稍微复杂一些就是接近百万行。而客户端一个平台的研发大多数也就在 5 人左右，代码量多在 10 万行以内。 也就是说, 很可能你一个人干两个人的活, 没跑的.

C++待遇方面, 15-25W之间. 如果超过30W, 你就可能不仅仅承担服务器端的工作了. (低于10W的, 说明你不符合要求, 然后企业降级录用的你), 其他不清楚.

有人说市场饱和? 我想你是招聘会去少了, 没见到抢人的吧.

技能点

后端开发工作, 对你的操作系统, 网络, 数据结构要求比较高哦. (业务知识是入行以后学习的, 这个反而好说)

有人想的很简单, 以为刷完APUE就可以了; 你去看看大厂的面试就知道, 融会贯通是需要知道很多知识然后整合的.
