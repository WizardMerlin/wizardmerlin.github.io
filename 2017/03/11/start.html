Markdown Rules Limits

1.  level : allow but limit (using "#" "##" "###", that is using level 1, 2 and 3)
2.  bold : forbid
3.  highlight : allow (only use it for key word)
4.  list : allow (welcome, use it as u like)
5.  link : allow (that is ok, just ok)
6.  footnode : allow (welcome, use it as u like)
7.  pics : allow (welcome, use it as u need)
8.  reference : allow (refer to books and famous people) 
9.  code block : allow (welcome, use it as u like)
10. italic: forbid
11. table: allow (that is ok, use it as u need)

Style Limits

1. one space between `char and number`, `char and punctuation`
2. do not use duplicate punctuation at the sametime.
3. name of `file.md` must be lower case (using `- ` instead of `_`)
4. paragraph should eat the line head

Launguage Limits

using english as much as possible.

Category Limits

1. codings (basic, language)
2. architectures (design)
3. managements (teamwork/leading)
4. domain knowledge (行业知识/领域知识)
5. books (read/ recomend)
