学习C++的利弊
post time: 2017-03-26
cpp 思考

每次在群里, 总有上学的同学, 一般是大二的, 为我现在C++行情如何? 要不要学? 还是学Java?

“奇怪，学习某项东西，是你的事情，跟我有什么关系? 再说了，我为什么要给你建议？我能拿到什么好处呀。”

当然，不是这样的。

不劝你学

我不劝你学习C++的原因其实很简单：
(排序分先后，必须分先后)

  • 学习某样东西第一要处于自愿：自己的兴趣爱好/自己乐意，这是最重要的。
  • 学习某项的技能一定要落到实处，要么是为了解决你个人的问题，要么是为了解决工作的问题等。
  • “学习”不是一件轻松的事情，他有时候是顺着大脑来的，有时候却是强迫自己做自己不愿意做的事情。
  • C++绝对不是一门速成的计算机编程语言，不敢说他是最难的(貌似scala，lisp更难？)，但是绝对是非常不容易的(学习成本太大)
  • C++的适用范围没有变大，但是竞争却多了。你可以看到，现在很多go, python, lua等，在原本C++专属的领域开始慢慢侵入，可以想象的是，以当前C++这样的成长速度(他有时候还去抄袭java的某些特性)，它应该会逐渐从大众语言变称小众语言
  • 一种语言就是一个体系而不仅仅只语言本身，不巧的是C++的朋友们数量也很多(甚至包括汇编/编译器/链接器等)

IT技术更新迭代快, 现在已经有很多语言在C++的领域可以(部分)代替C++, 如果你以后决心要走产品, 走管理, 其实不学也行.

我的前辈, 以过来人的身份说, 其实销售整体比技术收入要高一些(如果你做不下来的话)

建议你学

“但是事情总有例外的一面” , 蝙蝠侠-猫女(偷走维恩少爷妈妈的项链那一幕)

股市中不是有这句话么：

当大家都看空的时候就是底。

通俗解释就是：大家都不看好的时候，就开始反转了。
物以稀为贵—–我没有劝你学习C++啊，这绝对不是劝你学习C++。

还有, 即便你不做底层编码, 比如写驱动, 写引擎, 写后端交易系统代码, 学习C++对于你对于计算机系统, 以及整个IT行业的认识都很有帮助.

参考书

干一辈子革命, 读一辈子 C++ 书

文章末尾不给你点儿图，感觉对不住你，希望不要吓到你(在下不算C++的高手，一般开发人员，而已)。

其实书有很多, 为了节省你的时间, 我还是给你列举一下, 我经历过: (也有我的师傅指点我的)

  • 《C++ Primer》
  • 《Effective C++》
  • 《More Effective C++》
  • 《Effective STL》
  • 《C++语言演化》 这本比较特殊, 帮助你对 C++ 的各种机制有深刻的认识

读这些书, 应该可以省下一些时间, 至于《Inside C++ Object Model》这类讲原理的书, 快速翻一下, 不看也罢, 就面试的时候吹吹水而已.
