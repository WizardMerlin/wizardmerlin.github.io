线程安全和可重入

先说一些核心概念：

  1. 异步(Asynchronous)：如果两个操作可以独立进行，称它们是异步的(就是常说的，不会阻塞等待)。
  2. 并发(Concurrency)：看起来同时执行，但其实是顺序执行。如多个线程或进程在单个CPU上的行为。
  3. 并行(Parallelism)：真正同时执行。如多个线程或进程在多个CPU上的行为。
  4. 线程安全(Thread-safe)：代码能从多个线程被调用而不破坏结果。
  5. 可重入(Reentrant)：代码被多次调用产生的结果是可预期的。

注意：

  1. 可重复(reentrant)和线程(thread-safe)有些区别；线程安全的一定就可重入么？至少我们可以这样说，非线程的一定不可重入。
  2. 不知道你注意到了没有，thread-safe性能可能会有一些损失，但是可重入就一般不会。—–可重入侧重去除全局/静态成分的依赖/访问。

补充： 可重入宏

在一个多线程程序里，默认情况下，只有一个errno变量供所有的线程共享。在一个线程准备获取刚才的错误代码时，该变量很容易被另一个线程中的函数调用所改变。类似的问题还存在于fputs之类的函数中，这些函数通常用一个单独的全局性区域来缓存输出数据。为解决这个问题，需要使用可重入的例程。可重入代码可以被多次调用而仍然工作正常。编写的多线程程序，通过定义宏_REENTRANT来告诉编译器我们需要可重入功能，这个宏的定义必须出现于程序中的任何#include语句之前。

_REENTRANT为我们做三件事情，并且做的非常优雅：

  1. 它会对部分函数重新定义它们的可安全重入的版本，这些函数名字一般不会发生改变，只是会在函数名后面添加_r字符串，如函数名gethostbyname变成gethostbyname_r
  2. stdio.h中原来以宏的形式实现的一些函数将变成可安全重入函数
  3. 在error.h中定义的变量error现在将成为一个函数调用，它能够以一种安全的多线程方式来获取真正的errno的值

然后编译的时候加上-D _REENTRANT。

线程和进程

线程和进程的关系(LWP)

Linux下使用进程或者线程，并没有多大差别(创建的代价差不多，可能最大的差别在于进程切换的开销)。但是在window上就不一样，你重新创造一个进程的代价太大，多以一般选择使用并且多用线程。线程也有PCB(但是不一样)，但是没有独立的地址空间(相当于原来的同一个地址空间里面，现在就有俩pcb了)。(内核怎么区分呢？ 内核只看PCB，他根据PCB信息来区分，所以它并不区分进程或者线程)分配资源的时候，是按照进程的个数而不是线程的个数进行分配创建线程使用的是clone() + CLONE_VM等;

线程号和LWP号

查看LWP号： ps -Lf （在火狐浏览器里面多打开几个标签就可以了） ps aux | grep firefox, ps -Lf pid. 此时可以查看LWP
(线程号没有重复(不同于在进程中区分其他线程的tgid)，线程号tid是用来在同一个进程中区分不同的线程号的；linux内核依据LWP号来进行调度, 分配时间仑片)

线程共享问题

三级映射：

pcb(进程)或者进程的数据--->页目录(目录项集合)--->页表(页表项集合)---->物理页面(内存单元)----
    ----->MMU-----...--->....---->另外一个pcb(进程)

(MMU左边儿，3个箭头，3级映射)

同理，某一个进程创建出来的线程，和原来的进程，指向相同的页目录项。即线程虽然和原来的进程pcb不一样，但是三级也表达的指向却是一样的(但是内存空间不是完全相同的，代码段就不同/调用情况页不一样；即栈独立)物理页面(即共享数据)

调用的时候，栈基指针不变，栈顶指针往下移动，被调用的函数入栈，形成栈帧(局部变量/临时值)，下一次调用，即产生新的栈帧之前，要保存上一次的栈顶栈基指针。(每个栈帧都有ebp,esp)—–栈帧不断地向低地址扩展(可以简单想象称栈往下不断的添加像一个个方框的栈帧)。

补充：Kernel里面Thread Scheduler直接调度的 Kernel进程/LWP。

线程之间共享/非共享

线程共享哪些资源？

  1. 打开的文件描述符
  2. 信号的处理方式(由于线程是晚于信号的设计, 线程和信号兼容不是太好，并且信号本身处理起来就很复杂，最好不要搅和在一起)
  3. 当前的工作目录
  4. 凭证uid,gid
  5. 内存地址空间(除了栈意外的.txt,.data,.bss,heap，共享库)

(相当于，0-3G的进程地址空间都是共享的)

线程间不共享？

  1. 线程id
  2. 处理器临时存储的寄存器值，栈
  3. 独立栈空间(用于函数调用)
  4. errno(它不是全局变量么？)
  5. 信号屏蔽字
  6. 调度的优先级

线程的优缺点

优点：

  1. 程序的并发性，相应的提高了(理论上说该进程获得cpu时间增多了)
  2. 共享数据方便/通信也比较方便 (最简单的例子，全局变量直接拿着用了)
  3. 开销比较小

缺点：

  1. 稳定性不好： 进程操作时，使用的大多数稳定的系统调用；而线程此处使用的第三方库(虽然得到了内核的支持)
  2. 调试困难
  3. 对信号的支持没有进程好

优点突出，缺点可以忍受—–选择线程吧。

也就是本进程内，请选进程；多进程间，包括跨越本机，请选用socket
